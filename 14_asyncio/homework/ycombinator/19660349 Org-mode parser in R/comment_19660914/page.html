scotttrinh commented Apr 14, 2019

I've been working slowly on a similar project to write an org-mode parser in Rust and have been talking with @scarab5q about what we would want to do to get this project off the ground when he stumbled across this repo. What serendipity, right?

We made a Github org called org-rs and would be happy to house this repo there. I'd love to help with anything that feels unfinished in your current implementation. I am interested in writing a more idiomatic Rust version of the parser alongside your reimplementation of the elisp version, but think what you have here is a huge leap forward towards my actual interest in writing multi-platform applications on top of org. Having this implementation would be helpful anyway, as it presents a stable public-facing API that our implementation could be compatible with and have a baseline to benchmark any improvements in speed or size that our current work might offer.

If you're interested, let me know, and I'll rename org-rs/org-rs and fork this repo to the org-rs org and make you an admin on that org. Otherwise, to reduce confusion, I might just rename our separate project. Either way, I'm excited to have been shown this work. I look forward to helping and collaborating on this! Thanks for all of the hard work you've put into this thus far!
